現在のブランチにコミット済みの作業分をもとに、GitHub のプルリクエストを `gh` CLI で作成してください。

## 手順

1. まず現在の状態を確認する。
   - `git rev-parse --abbrev-ref HEAD` で現在のブランチ名を取得する。
   - `git status` で未コミットの変更が残っていないか確認する。**未コミットの変更があれば、勝手にコミットせず、ユーザーに確認する。**
   - `git log main..HEAD --oneline` で main との差分コミット一覧を取得する。差分が無ければ PR は作らず、その旨を伝える。
   - `git diff main...HEAD --stat` で変更ファイルの概要を把握する。

2. ブランチをリモートに push する（未 push の場合）。
   - `git push -u origin <ブランチ名>`

3. PR のタイトルと本文を組み立てる。
   - **タイトルは Conventional Commits 形式**（`<type>: <subject>`、スコープなし）。`<type>` は英語、`<subject>` は日本語。差分コミットの内容から一目で分かる簡潔なタイトルを付ける。
   - 差分コミットと変更内容から「概要」「変更内容」を具体的に記載し、「変更の種類」の該当項目に `[x]` を入れる。分からない項目（関連 Issue など）は空欄やテンプレートのコメントを残す。

4. `gh pr create` で作成する。ベースは `main`。
   - 本文はファイルに書き出して `--body-file` で渡すか、`--body` で渡す。
   - `--base main --head <ブランチ名> --title "<タイトル>"` を指定する。

5. 作成した PR の URL をユーザーに伝える。

## 注意

- 既に同じブランチの PR が存在する場合（`gh pr view` で確認）は新規作成せず、その旨を伝える。
- ユーザーが `$ARGUMENTS` でタイトルや補足を指定していれば、それを優先して反映する。
